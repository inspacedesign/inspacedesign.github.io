================================
照 片 命 名 與 整 理 規 則
================================

【資料夾對應】
每個案例一個資料夾，例如：
  images/case-01/   → 對應 cases/case-01.html
  images/case-02/   → 對應 cases/case-02.html
  ... 以此類推到 case-10/

【每個資料夾內必備】
  cover.jpg          ← 首頁縮圖＋案例詳細頁封面（最美的一張）
  01.jpg             ← 案例詳細頁的第 1 張照片
  02.jpg             ← 第 2 張
  03.jpg
  ...

【命名嚴格規則】
  ✅ 全部用英文＋數字
  ✅ 副檔名統一 .jpg（除非透明背景才用 .png）
  ✅ 數字補零兩位（01、02，不要 1、2）
  ❌ 不要中文檔名
  ❌ 不要空格（用 - 或 _ 取代）
  ❌ 不要奇怪符號

【建議照片規格】
  尺寸：寬度 1920px 以內（手機看也夠清楚了）
  大小：每張 < 500KB（網站才會跑得快）
  格式：JPG（檔小、相容性好）

  ⚠️ 如果是 Facebook 下載的圖，通常已經被 FB 壓縮過，
  尺寸可能太小或太模糊。建議用原始檔案或重新壓縮。

【Mac 壓縮圖片的方法】
  方法一（最簡單）：
    1. 在 Finder 選取所有要壓縮的圖
    2. 點兩下用「預覽程式」開啟
    3. 檔案 → 輸出（或 Cmd+Shift+S）
    4. 格式選 JPEG，品質滑桿往中間調
    5. 存檔

  方法二（批次處理）：
    用 Automator 建立批次壓縮工作流程
    （之後需要可以再請 Claude 教）

【整理完成後告訴 Claude】
  「照片整理好了」
  → Claude 會檢查每個資料夾的命名是否正確
  → 然後自動 push 到 GitHub
  → 網站幾分鐘內自動更新
